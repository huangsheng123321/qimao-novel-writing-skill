# 用户确认的新版流程

## 默认启动方式

用户不是先说“加载技能”，而是直接提出类似需求：

```text
请帮我查找番茄男频小说最近一个月的爆款小说（阅读量、完读量），分析该小说的整体框架是什么样子的，钩子是怎么设计的，高潮是怎么设计的等等。

然后给我 3-5 个仿写建议，并推荐一个，问我确认可以不。
```

## 严格状态机

### S0：爆款调研与结构分析

触发：用户要求查找番茄男频最近一个月爆款小说，分析阅读量/完读量、框架、钩子、高潮。

动作：

1. 查找/参考番茄男频近一个月爆款小说。
2. 分析阅读量、完读量、完读率等数据可见性。
3. 如无法确认真实数据，明确说明不能确认，不编造。
4. 拆解整体框架、黄金三章、开篇钩子、章节钩子、高潮设计、爽点设计、人设设计、冲突升级。
5. 进入 S1。

### S1：3-5 个仿写建议

动作：

1. 给 3-5 个仿写建议。
2. 推荐 1 个。
3. 结尾必须问：

```text
我建议优先选【方案X】。你确认可以不？
```

用户回复：

- `确认` → 进入 S2。
- `不行` / `重新推荐` / `换一批` / `不满意` → 留在 S1，重新给 3-5 个仿写建议。

### S2：主题方案、框架方案、仿写方案三案

动作：

1. 输出主题方案。
2. 输出框架方案。
3. 输出仿写方案。
4. 建议采用哪一个或如何组合。
5. 结尾必须问：

```text
我建议采用【X】为主，【Y】为辅。你确认可以不？
```

用户回复：

- `确认` → 进入 S3。
- `不行` / `重新给方案` / `重新推荐` / `不满意` → 留在 S2，重新给主题/框架/仿写三案。

### S3：整本小说大纲、人物角色、细纲和正文规划

动作：

1. 写整本小说大纲。
2. 写人物角色。
3. 写整本/第一卷细纲。
4. 写正文规划。
5. 结尾必须问：

```text
以上大纲、人物、细纲和正文规划，你确认可以不？
```

用户回复：

- `确认` → 进入 S4。
- `不行` / `重新写大纲` / `重新设定人物` / `重新细纲` → 留在 S3，根据意见重做。

### S4：写 5 章小说内容

动作：

1. 写 5 章小说正文。
2. 写完 5 章后不要自动审核。
3. 结尾必须问：

```text
前 5 章已经写完。你回复“确定”后，我先审核，再修改，最后给你一版可直接使用的最终稿。
```

用户回复：

- `确定` → 进入 S5。
- 其他修改意见 → 按意见调整正文，但不进入审核终稿阶段。

### S5：审核、修改、最终稿

动作：

1. 先输出五章审核报告。
2. 再输出具体问题表。
3. 再输出修改策略。
4. 最后输出修改后最终稿。

要求：不能只给建议，必须给可直接使用的最终稿。

## 关键词判定

- 只有“确认”才能推动 S1→S2、S2→S3、S3→S4。
- 只有“确定”才能推动 S4→S5。
- “可以”“行”“继续”可以视为弱确认，但如果流程严格执行，优先提醒用户回复“确认”。
- “不行”“重新推荐”“换一批”“重新给方案”“不满意”必须回到当前阶段重新输出。
